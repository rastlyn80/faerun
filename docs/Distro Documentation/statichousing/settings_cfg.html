Static Housing Documentation

For the sysbook package in the Distro, by Yukiko.

Static Housing config file settings

These settings can be found in the settings.cfg file in the config directory in the statichousing package.
PriceEditable Setting this to 1 will allow the staff GM deeding a house to edit the price per square in the final gump after using the staticdeed command. Setting it to 0 will not allow the price per square to be edited.
SecuresEditable Setting this to 1 will allow the staff GM deeding a house to edit the amount of secure containers allowed for a house in the final gump after using the staticdeed command. Setting it to 0 will not allow the amount of secure containers allowed for a house to be edited.
LockDownsEditable Setting this to 1 will allow the staff GM deeding a house to edit the amount of lock downs allowed for a house in the final gump after using the staticdeed command. Setting it to 0 will not allow the amount of secure containers allowed for a house to be edited.
RefreshTime At the time that a house is purchased and each time it is refreshed, by either an owner or co-owner using the sign or entering the house and walking through the doors, this is the amount of time added to the decay timer.
The time is in real life days.
HouseValuesCount Some server developers may wish to impose a limit on the number of houses an account or a character may own.
The value of a house might be based on two things (See the next setting).
HouseValuesDeterminedBy The two factors for limiting the number of houses owned are by Price and HousesOwned.
Options are:
Price - The value of a house will be determined by the integer value of price/HouseValueDivisor

HousesOwned - The house value will be determined by how many houses are owned.

With the setting HousesOwned, each house counts as 1 toward the TotalValueOfHouses CProp
set on either the account or the character which is determined by the HouseValueAppliedTo setting.
HouseValueAppliedTo Options are:
Account - If Account then no account may have more than MaxHouseValue on their account.

Character - If Character then no character may have more than MaxHouseValue on that character.
HouseValueDivisor This setting is used by the system only when the HouseValuesDeterminedBy setting is set to Price.

If house values are determined by Price then this will be the number divided into the price of the house at the time of purchase. You will have to adjust this based upon your pricing structure for houses. Some trial and error testing may be necessary to get the HouseValueDivisor to work out based on your pricing structure.
If the house value/HouseValueDivisor is less than 1 then it will default to 1.
MaxHouseValue This can be any integer you wish to assign. You will probably have to adjust this depending on whether you are determining house value by houses owned or your house pricing scale, and whether the house value applies to accounts or characters.
PricePerSquareDeterminedBy This setting tells the system how to determine the price per square to charge for a house. Default is the only option currently available.
Default - The DefaultPricePerSqure setting will be used if this is chosen.
See the settings.cfg file for the other two settings which are not yet implemented.
DefaultPricePerSquare This is the default price per square a player will be charged for a house.
This can be any integer from 1 up to whatever the developer chooses.
LockdownDivisor LockdownDivisor is divided into the total square footage area to determine the maximum number of lockdowns for a structure. The formula is: (x * y * z) / LockdownDivisor.
Where x and y are the dimensions of the level and z is the number of levels.
MinLockdowns MinLockdowns is the minimum lockdowns that can be assigned to a structure.
If the total lockdowns is less than this then this will be the minimum default.
SecureDivisor SecureDivisor is divided into the total cubic area to determine the number of secures for a structure.
The formula is: (x * y * z) / SecureDivisor.
Where x and y are the dimensions of the level and z is the number of levels.
MinSecures MinSecures is the minimum secures that can be assigned to a structure.
If the total secures is less than this then this will be the minimum default.
